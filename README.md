# Управление социальными проектами

## Описание проекта
Данный проект представляет собой систему автоматизации управления жизненным циклом социальных проектов на основе Telegram-бота и интерактивного дашборда аналитики. Система позволяет оптимизировать процессы обработки инициатив, отслеживания статусов и анализа социального эффекта.

## Цель проекта
Разработка комплексного решения для автоматизации управления социальными проектами, обеспечивающего прозрачность процессов, контроль сроков реализации и наглядную оценку загрузки исполнителей.

Основные характеристики
- **Telegram-бот** для управления проектами в реальном времени
- **Интерактивный дашборд** с визуализацией ключевых показателей
- **Автоматическое обновление** данных каждую минуту
- **CSV-база данных** для хранения информации о проектах
- **Многопользовательский интерфейс** с фильтрацией по исполнителям

## Предварительные требования
Java JDK 17 или выше
Python 3.8 или выше
Maven (для сборки Java-части)
Активный Telegram-бот (создается через @BotFather)

###Установка зависимостей

Java (Бот):
Зависимости подтягиваются автоматически через Maven (файл pom.xml).
Основная библиотека: telegrambots

Python (Дашборд):

pip install dash pandas plotly


Запуск

Запуск Telegram-бота:
Запустить класс SocialProjectBot.java в IntelliJ IDEA.

Запуск дашборда аналитики (в отдельном терминале):

python dashboard.py


Открыть дашборд в браузере: http://127.0.0.1:8050/

Синтаксис команд для Telegram-бота

Основные команды:

/start - начать работу с ботом

/help - получить справку по командам

Управление проектами:

Добавление нового проекта: /add Название, Проблема, Инициатор, Сроки
Пример: /add Парк Победы, Экология, Иванов И.И., 2025-05-20

Обновление информации о проекте: /update номер_проекта, поле, значение
Примеры:
/update 1, deadline, 2025-06-01
/update 1, problem, Урбанистика
/update 1, initiator, Петров П.П.

Удаление проекта: /delete номер_проекта
Пример: /delete 1

Просмотр списка проектов: /list

Детальная информация о проекте: /info номер_проекта
Пример: /info 1

Изменение статуса проекта: /status номер_проекта, новый_статус
Пример: /status 1, В работе

Назначение исполнителя: /assign номер_проекта, ФИО_исполнителя
Пример: /assign 1, Сидоров С.С.

Доступ к дашборду: /dashboard

Доступные поля для обновления:

name - название проекта

problem - социальная проблема

initiator - инициатор проекта

deadline - сроки (формат: ГГГГ-ММ-ДД)

status - статус проекта

executor - ответственный исполнитель

Структура данных

Файл projects.csv содержит следующую структуру:

ID - идентификатор проекта

Name - название проекта

Problem - социальная проблема / категория

Initiator - инициатор (автор идеи)

Deadline - срок реализации

Status - текущий статус (Новый, В работе, Завершен)

Executor - ответственный исполнитель

Особенности работы

Автоматическое сохранение - все изменения автоматически сохраняются в CSV-файл

Валидация данных - система проверяет наличие обязательных аргументов

Реальное время - дашборд обновляется каждую минуту, подтягивая свежие данные из бота

Фильтрация по исполнителям - возможность просматривать аналитику по конкретным ответственным

Технологический стек

Backend (Bot): Java 17+, TelegramBots API

Analytics (Dashboard): Python 3.8+, Dash, Plotly

Data: CSV, Pandas
